## 典型工作流

### 工作流 1: 新项目探索

```
1. 打开终端，进入项目目录
2. 运行 crush
3. "请分析一下这个项目的架构"
   → AI 使用 glob, grep, view 浏览项目
   → 返回架构概览和关键文件

4. "详细解释 internal/agent/coordinator.go 的逻辑"
   → AI 读取文件并解释核心逻辑

5. "画出这个系统的架构图"
   → AI 分析后生成 Mermaid 图
```

### 工作流 2: Bug 修复

```
1. "程序报了这个错误: [粘贴错误信息]"
   → AI 分析错误，搜索相关代码
   → 定位问题文件和行号

2. AI 建议修复方案
   → 展示需要的代码变更

3. 用户确认修复
   → AI 使用 edit 工具修改文件

4. "运行测试验证修复"
   → AI 执行 bash 运行测试
   → 报告测试结果
```

### 工作流 3: 功能开发

```
1. "添加一个新功能: [描述需求]"
   → AI 分析现有代码
   → 确定需要修改的文件

2. AI 逐步实现:
   → 创建/修改文件
   → 添加测试
   → 运行验证

3. "检查一下有没有安全问题"
   → AI 审查修改的代码
   → 报告潜在风险
```

### 工作流 4: 代码重构

```
1. "重构这个模块，提高可读性"
   → AI 分析代码结构
   → 提出重构方案

2. 用户确认方案
   → AI 逐步执行重构
   → 每步展示变更

3. "运行测试确保重构没有破坏功能"
   → AI 执行测试套件
```

## 配置场景

### 场景 1: 使用自定义 OpenAI 兼容 API

```json
{
  "providers": {
    "my-provider": {
      "type": "openai",
      "base_url": "https://api.my-service.com/v1",
      "api_key": "sk-xxx"
    }
  },
  "models": {
    "large": {
      "provider": "my-provider",
      "model": "my-model-name"
    }
  }
}
```

### 场景 2: 添加 MCP 服务器

```json
{
  "mcp": {
    "database": {
      "type": "stdio",
      "command": "mcp-server-sqlite",
      "args": ["--db", "myapp.db"]
    }
  }
}
```

### 场景 3: 配置 Hook 自动化

```json
{
  "hooks": {
    "pre_tool_use": [
      {
        "matcher": "bash",
        "command": "scripts/check-safety.sh",
        "timeout": 5000
      }
    ]
  }
}
```

### 场景 4: 配置 LSP

```json
{
  "lsp": {
    "go": {
      "command": "gopls"
    },
    "python": {
      "command": "pyright-langserver",
      "args": ["--stdio"]
    }
  }
}
```

## 最佳实践

### 1. 提示编写
- 明确描述目标，不要模糊
- 提供足够的上下文（文件路径、错误信息）
- 一次一个任务，不要一次要求太多

### 2. 会话管理
- 不同任务用不同会话
- 重要会话及时保存标题
- 长对话注意上下文压缩

### 3. 权限管理
- 使用 Hook 预批准安全操作
- 谨慎使用 "Allow Always"
- 定期审查权限设置

### 4. 模型选择
- 复杂推理用大模型
- 简单任务用小模型节省成本
- 根据项目需要选择合适的 Provider

### 5. MCP 扩展
- 按需添加 MCP 服务器
- 使用 `allowed_mcp` 控制工具范围
- 监控 MCP 服务器状态
