# 01 - 产品视角分析

## 产品定义

Crush 是一个**终端原生的 AI 编程助手**，由 Charm 团队（以终端美化生态闻名）开发。它不是一个 Web 应用或 IDE 插件，而是直接运行在终端中的独立工具。

## 核心价值主张

### 1. 终端原生体验
- 无需离开终端即可获得 AI 辅助
- 利用 Charm 生态（Bubble Tea、Lipgloss、Glamour）提供媲美 GUI 的终端体验
- 支持鼠标交互、语法高亮、Markdown 渲染、动画效果

### 2. 多 LLM 无缝切换
- 不绑定单一 LLM 提供商
- 支持 OpenAI、Anthropic、Google、Bedrock、Azure、OpenRouter 等
- 用户可按需切换，不被供应商锁定

### 3. 开发工具深度集成
- 63+ 内置工具（bash、文件编辑、搜索、Web 等）
- LSP 集成提供代码智能
- MCP 协议支持外部工具扩展
- Hook 系统支持自定义工作流

### 4. 会话持久化
- SQLite 后端保证会话不丢失
- 多项目管理、多会话并行
- 上下文自动压缩，长对话不中断

## 核心使用场景

### 场景 1: 代码探索与理解
```
用户 → 打开终端 → crush → 描述需求 → AI 读取项目 → 给出解释/建议
```
Agent 会自动使用 grep、glob、view 等工具浏览代码库。

### 场景 2: 代码编写与修改
```
用户 → 描述修改需求 → AI 分析文件 → 调用 edit/write 工具 → 展示变更
```
支持精确的字符串替换、多文件编辑，有文件变更追踪。

### 场景 3: 调试与修复
```
用户 → 粘贴错误信息 → AI 运行诊断 → 分析日志 → 提出修复方案
```
Agent 可执行 bash 命令、读取日志、运行测试。

### 场景 4: 自动化工作流
```
用户 → 配置 Hook → 触发条件 → 自动执行脚本
```
通过 Hook 系统实现 CI/CD 集成、自动格式化、安全检查等。

### 场景 5: 多模型协作
```
用户 → 大模型处理复杂推理 → 小模型处理简单任务 → 优化成本和速度
```
Large/Small 双模型架构自动选择。

## 产品差异化

### 技术差异化
1. **Go 原生性能**: 单二进制，无 Node.js 运行时依赖，启动快
2. **Charm 生态**: 终端 UI 美化领域的领导者在 TUI 体验上有天然优势
3. **Fantasy 抽象层**: 统一的 LLM Provider 接口，扩展新 Provider 成本低

### 体验差异化
1. **终端美学**: 渐变 Logo、动画效果、主题系统
2. **Mouse-friendly**: 不像传统 TUI 只依赖键盘
3. **响应式布局**: 自适应终端大小，紧凑模式自动切换

### 生态差异化
1. **MCP 协议**: 与 Claude 生态兼容的工具扩展
2. **Skills 系统**: 社区驱动的 Agent 能力扩展
3. **Hook 系统**: 用户自定义自动化

## 商业模式分析

Crush 采用开源模式，价值链在于：
- **开源核心**: 建立社区和用户基础
- **Provider 生态**: 通过 Fantasy 抽象层降低用户迁移成本
- **企业需求**: 自托管、数据安全、多模型选择满足企业合规

## 产品局限与挑战

1. **终端限制**: 无法显示图片、复杂 UI，部分场景不如 IDE 插件
2. **LLM 依赖**: 产品体验受限于 LLM 能力和 API 稳定性
3. **学习曲线**: 终端工具对非开发者不友好
4. **竞争激烈**: Claude Code、Cursor 等竞品有强大的模型和分发优势
