# 05 - UI 设计分析

## 设计理念

Crush 的 UI 建立在 Charm 团队的终端美学理念之上：**终端不应该是丑陋的**。利用 Bubble Tea v2、Lipgloss v2、Glamour v2 三件套，实现了媲美桌面应用的终端体验。

### 响应式设计

终端大小变化时自动适配：
- **宽终端**: 完整布局，显示所有信息
- **窄终端**: 紧凑模式，隐藏次要信息
- **极小终端**: 最小化布局，只保留核心交互

## 消息渲染系统

### 消息类型与视觉表现

| 消息类型 | 渲染方式 | 交互 |
|----------|----------|------|
| 用户消息 | 简洁文本 + 附件标签 | 无 |
| 助手文本 | Markdown + 代码高亮 | 可选中复制 |
| 助手思考 | 可折叠区域，默认收起 | 点击展开 |
| Bash 工具 | 命令行样式，行号 | 点击展开输出 |
| 编辑工具 | Diff 视图，增删高亮 | 查看变更 |
| 搜索工具 | 结果列表，匹配高亮 | 点击跳转 |
| MCP 工具 | 通用工具卡片 | 展开查看 |
| 错误消息 | 红色高亮 | 复制错误 |

### Markdown 渲染

使用 Glamour v2 渲染 Markdown，关键特性：
- **代码语法高亮**: 通过 Chroma 实现
- **自适应宽度**: 根据终端宽度自动换行
- **缓存渲染**: 同一内容不同宽度独立缓存
- **主题一致**: 代码高亮颜色与整体主题协调

```go
// 渲染器按宽度缓存
type rendererCache struct {
    renderers map[int]*glamour.TermRenderer
    mu        sync.RWMutex
}
```

### 主题架构

Crush 使用 Charm 的 charmtone 配色方案：

```

├── Header         → 头部样式
├── Messages       → 消息区域
├── Tool           → 工具调用

├── Status         → 状态栏
├── TextInput      → 输入框
├── Completions    → 补全弹窗
├── Attachments    → 附件区域
├── Diff           → Diff 视图
├── Markdown       → Markdown 内容
├── Logo           → Logo 渐变

```

### 视觉特效

- **渐变 Logo**: 使用 Lipgloss 渐变渲染
- **动画指示器**: Spinner / 进度条
- **悬停效果**: 鼠标悬停时高亮
- **选中效果**: 文本选中时背景色变化

### Textarea 组件

### 自动补全系统

```
触发: 输入 @ 或 / 或特定前缀
  → 文件补全: /path/to/file
  → 资源补全: @MCP 资源
```

### 附件系统

支持附加文件和图片到提示中：
- 文本文件: 读取内容作为上下文
- 图片文件: 以 Base64 编码发送给支持视觉的模型
- 拖放支持: 在支持的终端中可拖放文件

## 性能优化

### 渲染优化
- **缓存渲染结果**: 消息内容只渲染一次，后续直接使用缓存
- **惰性渲染**: 可见区域外的消息不渲染
- **动画节流**: 不在可见区域时暂停动画

### 滚动优化
- **视口计算**: 只渲染可见区域的消息
- **智能滚动**: 新消息时自动跟随，手动滚动时停止跟随
- **增量更新**: 只更新变化的消息，不全量重绘

### 内存管理
- **消息清理**: 离开会话时释放渲染缓存
- **图片处理**: 大图片压缩后再显示
- **资源释放**: 组件销毁时正确清理资源
